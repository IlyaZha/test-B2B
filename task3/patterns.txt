Класс User связан с классом Article связью Один-ко-многим (One-to-Many) - один пользователь может написать несколько статей.
Класс Article связан с классом User обратной связью Один-к-одному (One-to-One) - у каждой статьи может быть только один пользователь-автор.

В классе User применяется фабричный метод createArticle для создания нового объекта класса Article.
